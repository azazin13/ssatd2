# ssatd2
Система семантического анализа тендерной документации
вверху программы перечислены все библиотеки. Некоторые могут быть свёрнуты под словом импорт. Необходимо их всех добавить в интерпретатор. Для этого: название библиотеки копируем (то что идёт белым после слова импорт), в правом нижнем углу нажимаем Python 3.11, далее Interpreter settings, нажимаем + в правой части окна, вбиваем название библиотеки, слева снизу жмём Install package, ждём успешной установки.

для тессеракта необходимо: скачать его через установщик (прикреплённый), во время установки тыкнуть где-то в additional language data и ставим галку на русский. Путь к нему не теряем, он понадобится далее

скачиваем также второй прикреплённый файл

теперь путь к самому тессеракту и файлу нужно указать в программе. Добавить их в 34 и 35 строчке. После буквы r вместо моих старых путей.

осталось только также добавить пути к анкете и архиву с документацией в строчках 344, 345 (также вместо старых путей после буковки r вставить свои пути)

теперь можно запустить программу. Пути до файлов будут добавляться в столбец заполняется претендентом. Пока там может быть только один файл тк это старая версия (то есть если он найдёт несколько подходящих файлов старые сотрутся из ячейки. новой версии такого не будет, но её надо доделать, потом перекину)

по поводу заполнения анкеты

если поставить где-то лишний энтер перед тем как начать что-то писать или в середине, то программа сработает не корректно (может найдём способ это исправить, но пока надо аккуратно всё указывать)
порядок приоритетности значений и, или и тп: Сначала идёт разделение по большому «ИЛИ», затем по «;», которое обозначает и, и в конце по «или».
